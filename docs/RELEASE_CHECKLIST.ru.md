# Чеклист релиза TiltMerge

English version: [RELEASE_CHECKLIST.md](RELEASE_CHECKLIST.md)

## Уже подтверждено в репозитории

- Реализованы игровые режимы, онбординг, локальные сохранения, скины,
  достижения, локализация и локальные таблицы лидеров. Локализация покрывает 23 языка и автоматически применяет локаль устройства при первом запуске.
- A/B-когорты стабильно назначаются для каждой установки. `spawn_speed` меняет
  частоту спавна, а `onboarding_style` выбирает интерактивный онбординг либо
  вариант со слайдами.
- Android-пресеты включают вибрацию и доступ в интернет. Интернет нужен только
  для необязательных сетевых сервисов.
- CI проверяет скрипты, тесты, переводы и критичные для релиза настройки.

## Что требуется перед внешним релизом

1. Создайте Firebase Android-приложение для `com.akarakuts.tiltmerge`, не
   добавляйте `google-services.json` в Git, затем установите и проверьте плагин
   Godot Firebase.
2. Создайте ключ Play App Signing и передавайте данные подписи только через
   локальный `export_presets.local.cfg` или GitHub Actions secrets. Для сборки
   по тегу нужны `ANDROID_KEYSTORE_BASE64`, `ANDROID_KEYSTORE_PASSWORD` и
   `ANDROID_KEY_ALIAS`.
3. Соберите подписанный `.aab`, загрузите его в Internal Testing Play Console и
   проверьте установку, звук, вибрацию и наклон хотя бы на одном бюджетном
   устройстве.
4. Опубликуйте политику конфиденциальности по постоянному URL и заполните Play
   Data safety согласно фактически включённым в сборку SDK.
5. Если включена необязательная аналитика, проверьте в DebugView события
   `game_start`, `merge`, `game_over`, `daily_played` и `skin_selected`, затем
   настройте поэтапный rollout.
6. В Play Console загрузите локализованное описание витрины для каждой
   доступной локали (список 23 языков — в `translations/*.csv`).

## Условия допуска к релизу

- `./scripts/run_tests.sh` завершается успешно.
- `python3 scripts/validate_release.py` завершается успешно.
- Для точного коммита релиза пройден полный [чеклист Android QA](DEVICE_QA.ru.md).
- Все учётные данные внешних сервисов находятся только в защищённых локальных
  настройках или секретах CI.
