## 1. Обзор (One-Pager)

| | |
|---|---|
| **Название** | TiltMerge |
| **Жанр** | Hyper-casual / casual physics-merge puzzle |
| **Платформа** | Android (min API 24 / Android 7.0) |
| **Движок** | Godot 4.3+ (GDScript) |
| **ЦА** | 13–45 лет, любители коротких сессий (метро, очередь, перерыв) |
| **Сессия** | 30–90 секунд |
| **USP** (уникальное предложение) | Управление наклоном телефона + merge-механика с физикой |
| **Референсы** | Suika Game, Threes, 2048, Ballz |

### Pillar-цели дизайна
1. **Pick-up-and-play** — понять за 5 секунд, без туториала-эссе
2. **Tactile feel** — наклон и haptics дают «физическое» удовольствие
3. **Skill depth** — легко начать, сложно мастерить (позиционирование, комбо)
4. **Short loops** — смерть быстрая, рестарт мгновенный

## 2. Core Gameplay

### 2.1 Игровое поле
- Вертикальная «коробка» во весь экран (портрет 9:16)
- **Стены** слева/справа, **дно** — твёрдые, **потолок** = линия game over
- Физический мир: гравитация вниз, кубики — RigidBody2D

### 2.2 Кубики
- **Tier-цепочка (Suika-стиль):** один путь уровней 1→12; цвет = визуальная метка tier (палитра в BALANCE.md), не отдельная ось match.
- **Уровень (tier):** целое число ≥ 1.
- При слиянии **одинаковых по tier** → новый кубик уровня +1 (того же визуального пути).
- **Размер** растёт с уровнем (`base_radius × size_multiplier^(tier-1)`, см. `data/config.json`).
- Физика: масса, отскок (bounce), трение — подобраны под «пружинистое» чувство.

### 2.3 Управление наклоном
- **Первичный ввод:** акселерометр (`Input.get_gravity()` / `Input.get_accelerometer()`)
  - Наклон телефона влево/вправо → боковая компонента гравитации в физическом мире
  - Угол наклона маппится в боковую силу `tilt_force = clamp(angle * sensitivity, -max, max)`
- **Fallback:** свайп/касание по экрану задаёт целевой угол наклона (для девайсов без сенсора и для эмулятора)
- **Клавиатура (дебаг):** A/D или ←/→

### 2.4 Спавн
- Кубик появляется сверху по центру каждые `spawn_interval` сек
- **Цвет** выбирается из активной палитры (рандом со взвешиванием)
- **Сложность:** `spawn_interval` убывает со временем; после порога score начинают падать tier-2 кубики
- **Прогноз:** HUD всегда показывает tier следующего кубика. Игрок может заранее строить позицию и цепочки, а не только реагировать на физику.
- **Reroll:** в начале забега доступна одна бесплатная смена следующего кубика. Каждое комбо, кратное трём слияниям, возвращает один заряд (максимум три). Это награда за рискованную серию, а не платный расходник.

### 2.5 Слияние (Merge)
- На `body_entered` у двух кубиков: если `tier` равны:
  1. Удаляем оба
  2. Спавним кубик tier+1 в точке контакта
  3. **Событие merge:** частицы, haptic, +score, обновление комбо
- Кубик максимального tier при слиянии → «лопается» (большой score-бонус, эффект), второй не создаётся

## 3. Win / Lose / Progression

### 3.1 Game Over
- Любой кубик **касается потолка** дольше `game_over_grace` сек (≈ 1.5 сек) → Game Over.
- Grace-период даёт шанс вытолкнуть кубик вниз.

### 3.2 Счёт
- За каждое слияние: `base_score[tier] × combo_multiplier`
- `combo_multiplier` растёт за слияния в течение `combo_window` сек (≈ 2 сек)
- Комбо сбрасывается при промахе (нет слияния дольше окна)

### 3.3 Режимы (Phase 4)
| Режим | Описание |
|---|---|
| **Classic** | Бесконечный, game over по переполнению |
| **Blitz** | 60 секунд на максимум очков |
| **Zen** | Нет game over, медитативный темп |

### 3.4 Daily Challenge
- Детерминированный seed = `hash(date_string)` → одинаковая последовательность для всех
- Детерминированная цель дня: создать tier 4–6; за первое выполнение начисляется бонус к счёту
- Отдельный leaderboard

## 4. Meta & Удержание

- **Best score** — локально + (опц.) Google Play Leaderboard
- **Достижения** (10–20): «Первое слияние», «Комбо ×5», «Tier 7», «10 игр подряд»
- **Скины кубиков** — разблокировка за очки/достижения (косметика)
- **Daily Challenge leaderboard** — причина вернуться завтра
- (LiveOps, Phase 9) Сезонные events, limited skins

---

## 5. Распространение

TiltMerge — свободное ПО под **GNU GPLv3-or-later**. В игре нет рекламы,
внутриигровых покупок и платных преимуществ. Скины открываются только игровыми
достижениями и очками.

## 6. Управление (Controls)

| Действие | Ввод |
|---|---|
| Наклон влево/вправо | Акселерометр / свайп / A,D,←,→ |
| Пауза | Кнопка UI / Esc / Back |
| Рестарт | Кнопка на Game Over |
| Подтверждение (меню) | Тап |

### Accessibility
- Тоггл «свайп вместо наклона» (Settings)
- Тоггл haptics (Settings)
- Тоггл «reduce motion» (меньше screen-shake)
- Минимальный readable text size

## 7. ИИ / «Боты»
Не предусмотрены (single-player, лидерборды — асинхронные).

## 8. Уровни / Контент
Бесконечный режим (procedural спавн). Daily Challenge = процедурный по seed.
Контентные «уровни» не планируются в MVP.

---

## 9. Искусство / Аудио / Тема

- **Стиль:** минимализм, неон на тёмном фоне, мягкие тени
- **Палитра:** см. BALANCE.md §Палитра tiers
- **Аудио:** 1–2 спокойных lo-fi трека, SFX на merge/spawn/game-over
- **Haptics:** лёгкая вибрация на merge, сильная на game over, пульсирующая на комбо

---

## 10. Технический дизайн

- **Язык:** GDScript
- **Архитектура:** autoload-синглтоны (`GameManager`, `AudioManager`, `SaveSystem`, `Haptics`)
- **Физика:** встроенный 2D-физический движок Godot
- **Сохранения:** `user://save.json` (best score, скины, настройки)
- **Data-driven:** баланс в `data/config.json`, скины в `data/skins.json`
- **Локализация:** CSV string-таблицы `translations/en.csv`, `ru.csv`
- **CI:** GitHub Actions → APK debug на push
- **Аналитика:** опциональная, отключена без подключённого плагина

## 12. Риски и митигация

| Риск | Митигация |
|---|---|
| Акселерометр непредсказуем на части девайсов | Свайп-fallback обязателен с Phase 1 |
| Retention ниже цели на soft launch | Готовность итерировать core loop (Phase 3/4) |
| Задержка верификации Play Console | Завести аккаунт в Phase 1 (2–7 дней) |
| Performance на weak-девайсах | Тест на $80 смартфоне с Phase 2 |
| Отклонение в ревью (policies) | Privacy policy и Data safety заполнены по фактическим SDK |
